== Building Block View

=== Whitebox Overall System (Level 1)

The following diagram shows the decomposition of the Capri DAM into its primary
building blocks and their relationships to external infrastructure.

[plantuml, target="level-1-overview", format="svg"]
....

    component [Admin Portal\n(React + MUI)] as GUI <<React>>
    component [Rails Application\n(API + Background Workers)] as API <<Ruby on Rails>>
    database  [Metadata Store\n& RBAC] as DB <<PostgreSQL 14 + pgvector>>
    component [Background Workers\n(Sidekiq)] as Workers <<Redis>>
    component [CDN Config Service] as CDN <<Pluggable Adapter>>

GUI    <--> API     : REST / JSON\nGraphQL
API    <--> DB      : ActiveRecord\n(SQL + pgvector ANN)
API    <--> Storage : Upload / Sign
API    --> Workers  : Sidekiq jobs
Workers --> DB      : Metadata updates
CDN    <--> Fastly  : Provider API\n(Fastly / Cloudflare / Akamai)
Fastly --> Storage  : Fetch master\nasset
@enduml
....

==== Contained Building Blocks

[cols="1,3", options="header"]
|===
| Component | Responsibility

| **Admin Portal (React)**
| Web-based management UI for asset management, folder browsing, user & group
  administration, image/video profiles, metadata schemas, and ACL policies.
  Built with React 19 and Material UI v9.

| **Rails Application**
| The core orchestrator. Handles business logic, REST & GraphQL APIs, URL
  signing, permission evaluation, and cloud provider handshakes.
  Built with Ruby on Rails 8.1 / Ruby 4.0.3.

| **Metadata Store (PostgreSQL 14 + pgvector)**
| Relational database for persistent storage of asset records, technical
  metadata, RBAC (users, groups, folder policies), workflow state, and
  audit logs. The `neighbor` gem adds pgvector-backed columns for semantic/
  visual similarity search — there is no separate search service; both
  lexical (`ILIKE`/JSONB) and vector (cosine ANN) queries run against this
  same database.

| **Background Workers (Sidekiq)**
| Asynchronous queue for heavy or time-sensitive operations: media ingestion,
  metadata extraction, CDN purging, email delivery, and reports.
  Uses Redis as the queue backend.

| **CDN Config Service**
| Manages edge/CDN service configuration (VCL/rules, backends, cache-purge
  keys) via a pluggable adapter (`app/services/cdn_adapters/`): Fastly,
  Cloudflare, and Akamai adapters are implemented today, plus a no-op
  Null adapter for local/dev environments without a real CDN account.
|===

==== Contained Building Blocks (Level 2)

[cols="1,3", options="header"]
|===
| Component | Responsibility

| **API v1 Controllers**
| RESTful JSON endpoints consumed by the React frontend and external
  integrations: assets, collections, CDN configurations, metadata schemas,
  ingestion batches, and search.

| **Admin Controllers**
| System-level REST endpoints: user & group management, folder RBAC policies,
  image/video processing profiles, workflow management, email templates,
  and coverband coverage reports.

| **GraphQL Endpoint**
| Schema-first GraphQL API providing typed queries and mutations for asset
  management, user/group administration, and metadata retrieval.

| **RBAC Engine**
| Evaluates user permissions by walking the `UserGroup → FolderPolicy`
  closure tree. Implements the 7-column ACL matrix
  (read, modify, create, delete, replicate, manage, explicit_deny).

| **Asset Service**
| Orchestrates the full asset lifecycle: multi-cloud upload via ActiveStorage,
  Fastly IO URL construction with HMAC signing, focal-point-aware smart crop
  parameter injection.

| **Metadata Service**
| Handles EXIF/IPTC/XMP extraction during ingest, structured metadata schema
  validation, and bulk import/export (CSV/JSON).

| **URL Signing (Security Engine)**
| Generates HMAC-signed Fastly delivery URLs to prevent "denial-of-wallet"
  attacks and unauthorized parameter manipulation.

| **CDN Purge Service**
| Issues Fastly "Purge by Surrogate Key" API calls when assets are updated
  or deleted, ensuring instant global cache invalidation.

| **Sidekiq Workers**
| Async job queue with strict priority weighting, highest to lowest:
  `mailers`/`notifications` (weight 5) → `default`/`duplicate_detection`/`bin_purge`
  (weight 3) → `workflow`/`metadata`/`smartai`/`cdn_invalidation`/`edge_metadata`
  (weight 2) → `reports`/`ingest`/`downloads`/`edge_metadata_bulk` (weight 1).
  Prevents heavy bulk uploads, ZIP exports, and report generation from delaying
  urgent email/notification delivery. See `config/sidekiq.yml`.
|===

=== Level 2: Admin Portal (Whitebox)

The Admin Portal is mounted as isolated React components within Rails views
(Hybrid Monolith pattern). Each component is lazy-loaded via a `data-view`
attribute.

[cols="1,3", options="header"]
|===
| Component | Responsibility

| **Asset Browser (Folders)**
| Hierarchical folder tree with context menus, drag-and-drop asset
  organisation, and inline preview panel.

| **Asset Viewer / Editor**
| Full-page asset detail with metadata editor, focal-point picker overlay,
  version history, and collection membership.

| **Users Manager**
| Paginated data grid of all DAM users. Supports search, status toggle
  (active / suspended), password reset, and group assignment.

| **User Groups Manager**
| Tree view of the group hierarchy with a slide-in GroupOverlay drawer.
  Supports add/remove members, child-group nesting, and folder ACL editing
  via the Groups tab (parent can add/remove children; child view is read-only).

| **Image Profiles**
| CRUD management of named transformation presets (width, height, crop,
  format, quality) for the Fastly IO pipeline.

| **Video Profiles**
| Encoding profile management: codec, resolution, bitrate presets assigned
  to folder paths for automated video transcoding.

| **Metadata Schemas**
| Field-level schema designer for structured asset metadata:
  field type, validation rules, and folder-scope assignment.

| **ACL Matrix**
| Per-folder permission table rendered within both the Group overlay and the
  Folder panel. Columns map 1:1 to the 7 FolderPolicy ACL flags.

| **Folder Panel**
| Slide-in drawer showing folder-level statistics, assigned image/video
  profiles, and direct ACL policy editing.
|===
