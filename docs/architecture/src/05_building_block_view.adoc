== Building Block View

=== Whitebox Overall System (Level 1)
The following diagram shows the decomposition of the Headless DAM into its primary building blocks and their relationships to external infrastructure.

[plantuml, target="level-1-overview", format="svg"]
....
@startuml
package "Headless DAM System" {
    component [Admin Portal] as GUI <<React>>
    component [Core API] as API <<Node.js/Go>>
    database [Metadata Store] as DB <<PostgreSQL>>
    component [Search Index] as Search <<Typesense>>
}

cloud "Fastly Edge" as Fastly
cloud "Object Storage" as Storage <<S3/Azure/GCP>>

GUI <--> API : "REST/JSON"
API <--> DB : "SQL"
API --> Search : "Index Update"
API <--> Storage : "Upload/Sign"
Fastly --> Storage : "Fetch Master"
Fastly <-- API : "Purge/Config"
@enduml
....

==== Contained Building Blocks
[cols="1,3", options="header"]
|===
| Component | Responsibility
| **Admin Portal** | Web-based GUI for asset management, tagging, and visual focal-point selection.
| **Core API** | The orchestrator. Manages business logic, security (URL signing), and cloud provider handshakes.
| **Metadata Store** | Relational database for persistent storage of asset technical data and business tags.
| **Search Index** | High-performance index for "search-as-you-type" functionality across the asset library.
|===

=== Level 2: Core API (Whitebox)
Zooming into the **Core API** to show how it handles the "Headless" logic.

[plantuml, target="level-2-core-api", format="svg"]
....
@startuml
package "Core API" {
    component [Storage Service] as StorageSvc
    component [Metadata Service] as MetaSvc
    component [Transformation Engine] as URLGen
    component [Security Engine] as Sec
}

database "PostgreSQL" as DB
cloud "Object Storage" as Cloud

StorageSvc --> Cloud : "Store/Delete"
MetaSvc --> DB : "CRUD"
URLGen --> MetaSvc : "Get Params"
URLGen --> Sec : "Sign URL"
@enduml
....

==== Contained Building Blocks (Level 2)

[cols="1,3", options="header"]
|===
| Component | Responsibility
| **Storage Service** | Abstracted interface to handle multi-cloud uploads. It abstracts the differences between S3, Azure Blob, and GCP buckets.
| **Metadata Service** | Handles the extraction of EXIF/IPTC data during upload and manages business-level attributes.
| **URL Generator** | The "Transformation Engine Wrapper." It builds the specific Fastly IO query strings (width, height, crop) based on requested presets.
| **Security Engine** | Responsible for HMAC signing of delivery URLs to prevent unauthorized parameter manipulation.
|===

=== Level 2: Admin Portal (Whitebox)
The Admin Portal is a decoupled Single Page Application (SPA).

[cols="1,3", options="header"]
|===
| Component | Responsibility
| **Uploader** | Handles multi-part uploads directly to the API with progress tracking.
| **Focal Point Editor** | A visual tool allowing users to click an image to set X/Y coordinates for cropping.
| **Asset Browser** | Consumes the Search Index to display instant results with infinite scroll.
|===
