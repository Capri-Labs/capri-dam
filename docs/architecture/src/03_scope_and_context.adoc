== System Scope and Context

=== Business Context
The Capri DAM serves as the central hub for digital assets. It bridges the gap between creative production (uploads) and digital consumption (web/mobile).

[cols="1,2,2", options="header"]
|===
| Communication Partner | Input to Capri DAM | Output from Capri DAM
| **Content Creators** | Master assets (High-res), Metadata (Tags, Focal points). | Upload status, Asset previews, Search results.
| **Frontend Applications** | Asset IDs and required transformation parameters. | Optimized, transformed Image/Video URLs.
| **Storage Providers** | Storage confirmation and binary retrieval. | Asset binaries (Write/Read commands).
| **Edge/CDN Provider** | Request for original asset from origin. | Transformed versions of assets for end-users. Pluggable adapter (see `app/services/cdn_adapters/`) — Fastly today, with Cloudflare, Akamai, and a no-op Null adapter also implemented.
|===

=== Technical Context
This section describes the technical interfaces and protocols used for communication between the Capri DAM and its environment.

[cols="1,2,2", options="header"]
|===
| Interface | Protocol / Tool | Description
| **Management API** | REST / JSON | Used by the Admin UI to manage asset records, tags, and configurations.
| **Binary Storage** | S3 API / Blob API / Local disk | Used to store and retrieve the original "master" files — pluggable `StorageAdapters::*` (AWS S3, Azure Blob, GCS, or local disk for self-hosted/dev use).
| **Asset Delivery** | HTTPS (CDN) | The path through which end-users receive transformed assets via the active CDN adapter (Fastly IO by default).
| **Search & Similarity** | Internal (PostgreSQL) | Faceted metadata/lexical search (`ILIKE` + JSONB property filters) and pgvector cosine-similarity nearest-neighbour search over asset embeddings — both run inside the same PostgreSQL instance as the metadata store, no external search service.
| **Cache Control** | Fastly Purge API | Programmatic invalidation of cached assets when metadata or binaries change.
|===



==== Scope Boundaries
* **Inside Scope:**
** Asset metadata management.
** Orchestration of uploads to multiple cloud providers.
** Generation of signed Fastly URLs.
** Search, indexing, and discovery logic.
* **Outside Scope:**
** Persistent storage of binaries (delegated to S3/Azure/GCP).
** Image processing/transformation logic (delegated to Fastly IO).
** User Identity Management (delegated to OIDC/OAuth2 providers).
