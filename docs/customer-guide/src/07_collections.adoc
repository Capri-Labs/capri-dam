=== Collections

==== 1. Feature Overview

**What it does:** Collections let you curate cross-folder sets of assets — manually,
by rule, or by time window — and share them externally without granting access to
your underlying folder structure.

**The business value:** Build a campaign kit or brand portfolio once, and share it
safely with anyone, inside or outside your organisation.

image::collections.png[Collections, title="Placeholder — a curated Collection ready to share (screenshot coming soon)"]

==== 2. How It Works

**The trigger:** Creating a Collection, or defining a Smart Collection's matching
rule.

**What the system does:** For manual Collections, assets are added exactly as you
choose. For **Smart Collections**, a background process continuously matches new and
updated assets against your rule (content type, tag, folder scope, date range) and
adds them automatically. **Temporal Collections** refresh automatically as time
passes (e.g. "assets uploaded this week").

**The output:** A shareable Collection page, addressable at its own URL, optionally
exposed via a signed public link with no login required for the recipient.

==== 3. Step-by-Step Instructions

. Open **Collections** from the sidebar and click **New Collection**.
. Choose **Manual**, **Smart**, or **Temporal**.
. For a Manual Collection, add assets via drag-and-drop or the **Add Assets** search
  picker.
. For a Smart Collection, define your matching rule (content type, tag, folder, date).
. Click **Share** to generate a signed public link, and set its expiry.

==== 4. Options & Configuration

* **Rule editor** — Smart Collections support content type, tag, folder scope, and
  date-range predicates.
* **Share link expiry** — set how long an external share link remains valid.
* **Nested/slug-based routing** — Collections can be organised in a nested,
  workspace-style structure.

==== 5. Access & Permissions

**Roles required:** Creating and editing Collections requires Contributor role and
above; viewing a Collection you have folder access to is available to any signed-in
user.

**Restrictions:** External recipients of a shared link never need a Capri DAM account
and only ever see the curated Collection contents — never your source folder
structure or permissions.

==== 6. System Limits & Technical Requirements

* Smart Collection rules currently support a **fixed set of predicates** (content
  type, tag, folder scope, date range) — there is no arbitrary boolean rule builder.
* Shared Collection links support **expiry-based** access control only — there is no
  password-protection option yet.

==== 7. Troubleshooting & FAQs

**A new asset matching my Smart Collection's rule isn't showing up yet. Why?**
Smart Collections are updated by a background process shortly after a matching asset
is created or changed — give it a moment and refresh.

**Can I password-protect a shared Collection link?**
Not currently — only expiry-based access control is available today.
