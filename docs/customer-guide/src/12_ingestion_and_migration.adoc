== Seamless Migration & Ingestion

=== 1. Feature Overview

**What it does:** A guided, wizard-driven migration brings your assets over from your
existing platform — no engineering team required.

**The business value:** Moving to Capri DAM doesn't mean starting from zero — your
existing library comes with you, with clear visibility into progress the whole way.

image::ingestion-migration.png[Seamless Migration & Ingestion, title="Placeholder — the guided migration wizard (screenshot coming soon)"]

=== 2. How It Works

**The trigger:** An administrator starts a migration batch via the **Start Migration**
wizard.

**What the system does:** A connector authenticates against your source system (e.g.
Bynder, Adobe Experience Manager, SharePoint, Cloudinary), pages through its assets in
chunks, and stages each file into Capri DAM under your chosen destination folder.
Progress is tracked per batch, with phase and status visible throughout.

**The output:** Migrated assets appear in your chosen destination folder just like any
other upload — including automatic metadata extraction and duplicate detection.

=== 3. Step-by-Step Instructions

. Open the **Migration Dashboard** from the sidebar.
. Click **Start Migration**.
. Choose your **source connector** (e.g. Bynder, S3, SharePoint).
. Select a **destination folder** using the built-in search box.
. Confirm and start — track progress from the batch list, which shows status chips
  per batch.

=== 4. Options & Configuration

* **Batch status filtering** — filter the batch list by status (e.g. extracting,
  transforming, completed).
* **Manual refresh** — pull the latest batch status on demand.
* **Legacy Connectors screen** — configure and store source-system credentials ahead
  of starting a migration.

=== 5. Access & Permissions

**Roles required:** Starting a migration and configuring connectors requires Admin
permissions. Once migrated, assets follow the same folder-level permissions as any
other asset.

=== 6. System Limits & Technical Requirements

* **Supported source platforms:** AWS S3, Google Cloud Storage, Nuxeo, Brandfolder,
  Cloudinary, SharePoint, Bynder, Aprimo, Widen Collective, Adobe Experience Manager,
  FTP/SFTP, MediaValet, Extensis Portfolio, and Canto.
* **Large batches** (10,000+ assets) can take 30–90 minutes to fully extract and
  transform — this is expected, not a stall.
* The wizard currently supports choosing a **destination folder** but does not yet
  expose granular field-mapping/transformation rules in the UI itself.

=== 7. Troubleshooting & FAQs

**My migration batch seems stuck. What should I check?**
Very large batches can legitimately take 30–90 minutes or more. If the pending item
count hasn't moved in a much longer window, ask an admin to check the source
connector's credentials are still valid.

**Can I retry a failed batch?**
Yes — batch rows offer retry/cancel actions; if a batch is in a fully `failed` state,
start a new batch instead.
