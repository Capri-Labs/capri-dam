== AI-Powered Smart Uploads & Organization

=== 1. Feature Overview

**What it does:** When you upload an image, Capri DAM automatically reads any visible
text, extracts technical details, and generates a semantic "fingerprint" so the asset
is instantly findable by meaning, not just filename.

**The business value:** It eliminates manual data entry and ensures every asset is
searchable the moment it lands — no tagging backlog, ever.

=== 2. How It Works

**The trigger:** Processing starts automatically the instant your upload reaches 100%
— there is no separate "Process" button to click.

**What the system does:**

* **Text recognition (OCR)** — any visible text in the image (a sign, a label, a
  storefront name) is read and stored, so you can search for what the image *says*,
  not just what it's named.
* **Technical metadata extraction** — camera/EXIF details, dimensions, and colour
  profile are captured automatically.
* **Semantic embedding** — a vector "fingerprint" of the image's visual meaning is
  generated by your organisation's configured AI provider (OpenAI, Cohere, Ollama, or
  Azure OpenAI), which is what powers meaning-based search like *"people eating in a
  park."*

**The output:** Extracted text and technical metadata are saved to the asset's
metadata profile immediately; the semantic fingerprint is saved a short time later
(typically seconds), so the asset becomes semantically searchable without you needing
to do anything further.

=== 3. Step-by-Step Instructions

. Click **Upload** from any folder.
. **Select your file(s)** or drag them directly onto the browser.
. Watch the progress bar — once it reaches 100%, processing begins automatically.
. The asset appears in your library with a **Ready** status badge once processing
  completes (typically a few seconds for images).
. Open the asset and check its **Properties** panel to see the extracted text and
  technical metadata Capri DAM found.

=== 4. Options & Configuration

* **Manual metadata edits** — open any asset's Properties panel and edit or add
  metadata fields by hand at any time; manual edits are never overwritten by the
  automated pipeline on later saves.
* **Custom metadata schemas** — admins can define structured, folder-specific metadata
  fields (beyond free-text) so your organisation's own taxonomy is captured
  consistently — see the Asset Management chapter of the
  link:../product-info/[Product Info guide].
* **Upload restrictions** — admins can restrict uploads to an approved allow-list of
  file types organisation-wide, from *Tools → Asset Configurations → Upload
  Restrictions*.

=== 5. Access & Permissions

**Roles required:** Available to all authenticated users with upload permission on the
destination folder (Contributor role and above).

**Restrictions:** Viewer-role users can search and view extracted metadata but cannot
upload new assets. Editing a folder's metadata schema requires Manager/Admin
permissions on that folder.

=== 6. System Limits & Technical Requirements

* **Maximum file size:** 2 GB by default — admin-configurable per organisation from
  *Tools → Asset Configurations → Upload Limits*.
* **Text recognition (OCR):** runs on image formats only (e.g. JPEG, PNG, WebP, GIF,
  TIFF, PSD, HEIC/RAW). It does not run on video, audio, or plain document files.
* **Semantic embeddings:** quality and availability depend on your organisation's
  configured AI embedding provider; if no provider is configured, keyword and metadata
  search still work normally, but meaning-based search will not return results.

=== 7. Troubleshooting & FAQs

**What happens if text recognition or metadata extraction fails?**
The asset still uploads and becomes usable — its status becomes **Ready** regardless.
Only the specific metadata field that failed to extract is left blank; nothing blocks
the asset from being stored, viewed, or shared.

**Can I process PDF or document files this way?**
Text metadata is extracted for PDFs and searchable text documents, but object/scene
recognition and OCR are image-specific — they don't run against document or video
files.

**Why isn't my new upload showing up in a semantic search yet?**
Semantic fingerprinting happens moments after the technical processing step. Give it
a few seconds and search again; if it still doesn't appear after a few minutes, check
with your administrator that an AI embedding provider is configured.
