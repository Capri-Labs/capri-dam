[[section-overview]]
== What is Capri DAM?

**Capri DAM** is a modern, open-source, **headless Digital Asset Management** platform
designed for teams that need enterprise-grade dynamic media capabilities without
enterprise-grade licensing costs.

It is built on **Ruby on Rails 8.1** + **React 19** and exposes every feature through
both a **REST API** and a **GraphQL API**, making it suitable as the media backbone for
any frontend — web, mobile, IoT, or digital signage.

.Key differentiators
* **Bring Your Own Storage** — connect AWS S3, Google Cloud Storage, Azure Blob,
  Cloudflare R2, Wasabi, DigitalOcean Spaces, or Backblaze B2.
* **Edge-first delivery** — real-time image transforms (crop, resize, WebP, focal-point
  smart-crop) via Fastly Image Optimizer; ~90 % reduction in rendition storage costs.
* **Headless by design** — no lock-in to a specific frontend; consume assets via
  REST, GraphQL, or signed CDN URLs.
* **Apache 2.0 licensed** — host it yourself; no per-seat or per-GB fees.

[[section-features]]
== Feature catalogue

=== Asset management

[cols="2,3", options="header"]
|===
| Feature | Detail

| Upload & ingest | Drag-and-drop browser upload, direct-to-storage presigned PUTs, and bulk ingestion from 14 external sources.
| Folder hierarchy | N-ary tree with materialised paths; per-folder RBAC policies, metadata schemas, and image/video profiles.
| Version history | Immutable version snapshots on every asset replacement; diff view in the UI.
| Soft-delete / Recycle Bin | 30-day (configurable) retention; bulk restore; workflow-protected purge guard.
| Metadata | Technical (EXIF/IPTC/XMP) auto-extracted; custom schema-driven fields per folder; bulk CSV import/export.
| Duplicate detection | Checksum-based automatic detection; side-by-side resolution UI.
| Quarantine | Security anomaly detection flags suspicious assets and prevents delivery until reviewed.
|===

=== Search & discovery

[cols="2,3", options="header"]
|===
| Feature | Detail

| Full-text search | PostgreSQL `tsvector` across title, description, tags, and metadata.
| Faceted filtering | Content type, date range, tags, folder scope, status.
| Semantic search | pgvector cosine similarity using AI-generated embeddings; supports OpenAI, Cohere, Ollama, Azure OpenAI.
| Semantic cluster map | Interactive force-directed graph of semantically similar assets for visual discovery.
| AI Copilot | Natural-language chat interface that translates queries into structured search filters.
|===

=== Delivery & CDN

[cols="2,3", options="header"]
|===
| Feature | Detail

| Real-time image transforms | Width, height, crop mode, format (WebP/AVIF/JPEG/PNG), quality — computed at Fastly edge.
| Smart crop | Focal-point–aware crop preserves the subject across all device breakpoints.
| HMAC-signed URLs | Prevents unapproved transform parameters and denial-of-wallet attacks.
| Surrogate-key cache purge | Instant per-asset or bulk Fastly cache invalidation via the API or Admin UI.
| Video encoding | Configurable FFmpeg encoding presets with folder-path assignment; output stored in ActiveStorage.
| Image & video profiles | Named preset bundles assigned to folder paths; automatically applied on delivery URL generation.
|===

=== Workflows & approvals

[cols="2,3", options="header"]
|===
| Feature | Detail

| Visual workflow designer | DAG-based canvas (`@xyflow/react`) for building approval and automation flows.
| Step types | Start, Approval (human task), Notification, Auto-archive, Auto-publish, Condition branch, End.
| Trigger rules | Metadata-based predicates: folder path prefix, content type, custom field value.
| Task assignment | Assign tasks to users or groups; email + in-app notification; due date and escalation.
| Workflow-protected purge | Assets with active workflow instances are blocked from permanent deletion.
|===

=== Access control

[cols="2,3", options="header"]
|===
| Feature | Detail

| Role hierarchy | `viewer` → `contributor` → `manager` → `admin` → `super_admin`.
| Group-based RBAC | Closure-table group hierarchy; users inherit parent group permissions.
| 7-column folder ACL | Per group × folder: read, upload, edit metadata, download, delete, manage, share.
| Keycloak SSO | Enterprise SSO via OmniAuth/OIDC; users provisioned on first login.
| OAuth2 server | Doorkeeper for external application token issuance.
| Personal Access Tokens | Long-lived hashed tokens for CI/scripts.
| Impersonation | Admin can impersonate any non-admin for support; fully audited.
|===

=== Collections

[cols="2,3", options="header"]
|===
| Feature | Detail

| Manual collections | Curated sets; assets added by drag-drop or API.
| Smart collections | Rule-based auto-populated collections updated by `SmartCollectionRouterWorker`.
| Temporal collections | Time-windowed collections (e.g. "assets uploaded this week").
| Share collections | Generate a signed public URL for a collection (configurable expiry).
|===

=== Administration

[cols="2,3", options="header"]
|===
| Feature | Detail

| User & group management | Full CRUD in the Admin UI; group hierarchy editor.
| Email templates | Handlebars-based transactional templates editable in the browser.
| Storage backend management | Configure and switch storage providers without downtime.
| System status dashboard | Queue depths, storage usage, error counts, health indicators.
| Reports | Configurable report definitions; generate CSV/XLSX/PDF on demand or on schedule.
| Data migration | Migrate between storage providers with progress tracking and audit report.
| Metrics | Daily aggregated statistics (`DailyMetric`) for content, storage, and usage trends.
|===

[[section-use-cases]]
== Use cases

=== Creative agencies

Upload, organise, and share brand assets with clients via self-expiring signed URLs.
Smart collections automatically group assets by campaign tag.
Approval workflows route new assets through a creative-director sign-off before publication.

=== E-commerce / retail

Manage tens of thousands of product images. Fastly IO delivers perfectly sized,
format-optimised images to every device in <100 ms without pre-generating renditions.
Focal-point smart-crop keeps products centred on all breakpoints.

=== Media & publishing

Ingest from existing Adobe Experience Manager or Bynder libraries via built-in
ingestion adapters. Full EXIF/IPTC metadata extraction. Semantic search finds related
images by meaning, not just keywords.

=== Enterprise / regulated industries

Keycloak SSO, group-based RBAC, immutable audit logs, and workflow-guarded purge
satisfy strict information-governance requirements.
