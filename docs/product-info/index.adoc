[[section-welcome]]
== Welcome

This guide is a module-by-module tour of **Capri DAM** for customers, evaluators, and
new team members. Every module chapter follows the same structure so you can jump
straight to what you need:

[.feature-highlight]
====
**How to read each module**

* **Overview** — what the module does and why it matters.
* **Screenshot** — a real capture of the running application (see
  <<section-screenshots,how these are produced>>).
* **Key features** — the concrete capabilities, in a scannable table.
* **Powerful / notable capabilities** — the handful of things that set this module apart.
* **Notes & known limitations** — honest, current caveats — not marketing copy.
* **Test coverage status** — which user flows already have automated End-to-End (E2E)
  coverage today, and which are still on the backlog (see
  <<section-coverage,Test Coverage Roadmap>>).
====

[[section-screenshots]]
=== About the screenshots

Every screenshot in this guide is a **real capture of the running application**,
taken with a small Playwright driver script —
`docs/product-info/scripts/capture_screenshots.js` — the same automation technology
used by the project's E2E test suite (`spec/e2e/`). It logs in as a seeded admin user
and navigates to each screen, so what you see here is what the product actually looks
like, not a mock-up.

[TIP]
====
To refresh every screenshot after a UI change:

[source,bash]
----
bin/rails s -p 3000 &                                  # 1. start the app
node docs/product-info/scripts/capture_screenshots.js   # 2. re-capture all screens
----
====
