[[module-search]]
=== Search & Discovery

==== Overview

Beyond simple keyword lookup, Capri DAM combines full-text search, faceted
filtering, AI-powered semantic (meaning-based) search, and a natural-language
Copilot chat interface, all from a single global search bar available on every
screen.

image::search-results.png[Search results page, title="Search Results page — query and mode carried in the URL, results rendered in a responsive grid"]

==== Key features

[cols="2,3", options="header"]
|===
| Feature | Detail

| Global search bar | Present in the header on every page; mode dropdown (Keyword, Visual Match, Ask AI Agent); keyboard shortcut (⌘K); ArrowDown/Enter keyboard navigation of suggestions.
| Full-text search | PostgreSQL `tsvector` across title, description, tags, and metadata.
| Faceted filtering | Content type, date range, tags, folder scope, status.
| Semantic search | pgvector cosine-similarity search using AI-generated embeddings; supports OpenAI, Cohere, Ollama, and Azure OpenAI as embedding providers.
| Semantic cluster map | Interactive force-directed graph (`@xyflow/react`-style canvas) of semantically similar assets, for visual/exploratory discovery.
| Suggestions panel | Live-typing suggestions with a "View all results" link that carries the query into `/search`.
| URL-addressable results | `/search?q=...&mode=...&page=...&per_page=...` — shareable, bookmarkable, back/forward-button friendly.
| Results-per-page control | Dropdown for 25 / 50 / 100 results per page (default 25), matching the folder/asset grid's page-size control.
| Include Recycle Bin toggle | Off by default; when enabled, soft-deleted assets are included in results and flagged with a "Bin" badge so they're clearly distinguishable from active assets.
|===

==== Powerful / notable capabilities

[.feature-highlight]
=====

* **Meaning, not just keywords** — semantic search finds visually or conceptually
  related assets even when they share no common tags or filenames.
* **One search bar, three search paradigms** — keyword, visual similarity, and
  natural-language agentic search are all one dropdown away, without needing
  separate screens.
* **Cluster map for serendipitous discovery** — surfaces relationships between
  assets that faceted filters alone would never reveal.
=====


==== Notes & known limitations

* Semantic search quality depends entirely on the configured embedding provider;
  there is currently no built-in provider fallback/failover if the primary
  embedding API is unreachable.
* The semantic cluster map is best-suited to small-to-medium result sets; very
  large clusters (1000+ nodes) are not yet paginated or virtualised.

==== Test coverage status

[.status-covered]#Covered by E2E# (`spec/e2e/global_search_bar.e2e.spec.js`,
`spec/e2e/search_and_duplicates_fixes.e2e.spec.js`)

* Search bar visible with default placeholder; mode dropdown includes Visual Match
  and Ask AI Agent.
* Selecting Visual Match / Ask AI Agent changes the placeholder text.
* Typing a query shows a suggestions panel; ArrowDown highlights the first
  suggestion without navigating away.
* Pressing Enter, and clicking "View all results", both navigate to `/search`
  with the query preserved.
* `/search` results page loads with pagination params and renders a results grid;
  accepts query + mode from the URL directly.
* Results-per-page dropdown defaults to 25 and re-fetches with the selected value.
* Result size renders as a human-readable string (never a hard-coded "0 KB").
* "Include Recycle Bin" toggle is off by default; enabling it re-queries with
  `include_bin=true` and flags bin results with a "Bin" chip.
* Clicking a result navigates to `/assets?id=<uuid>` and keeps that param in
  the URL (previously stripped by the folder explorer's URL-sync effect).
