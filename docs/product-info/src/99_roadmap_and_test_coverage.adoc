| 📋 Planned | DAM-to-DAM federation | Cross-instance asset sharing via signed GraphQL federation.
| 📋 Planned | AI auto-tagging | Automatic tag suggestions from a computer-vision model at upload time.
| ✅ Done | Bulk copy (folders & assets) | "Copy to…" action mirroring the existing "Move to…" action — select any mix of folders/assets and copy them into a new destination in one request (`POST /api/v1/copy_operations`). Recursively duplicates folders, shares the source's ActiveStorage blob instead of re-uploading bytes, and auto-resolves name collisions.
| ✅ Done | Bulk/folder ZIP download | "Download" action on a folder or any multi-asset selection streams a ZIP built asynchronously by `AssetDownloadWorker`, with a live progress bar, queue-awareness (a second download while one is running is queued, not run concurrently), and the finished ZIP link delivered to the user's Inbox and notification bell.
| 📋 Planned | Manual static renditions | User-facing "Add/Delete rendition" action on an asset's Properties page (`renditions` table and model already exist; today renditions are worker-generated only — video MP4/OGG/AV1 — with no CRUD UI or endpoint).
| ✅ Done | Direct Publish REST action | `POST/POST /api/v1/assets/:id/publish`\|`/unpublish` — publishing an asset no longer requires building a workflow first. "Manage Publish" is a dedicated menu next to (not inside) *Tools* in the Explorer, offering immediate Publish/Unpublish plus "…Later" scheduling (`ScheduledPublishAction` + `PublishSchedulerWorker`, polled every 5 minutes) for a future date/time. Asset-only — never shown for a selection containing folders.
| ✅ Done | Contextual metadata rules (Choices / Requirement / Visibility) | Metadata Schema fields can depend on another `select` field's value, configured entirely via the Schema Editor UI (no code changes): **Choices** narrows a dependent `select`'s options (`field.rules.cascade`); **Requirement** dynamically marks any field type required (`field.rules.requirement`); **Visibility** shows/hides any field type (`field.rules.visibility`). Resolved client-side by `AssetMetadataPanel#resolveFieldRules`, with save-time validation across all tabs.
| ✅ Done | Custom "funny 404" error page | A shared `ApplicationController#render_not_found` (triggered by `ActiveRecord::RecordNotFound` and by a `config/routes.rb` catch-all for unmatched routes) replaces raw Rails diagnostics pages with an illustrated, on-brand 404 screen for browser navigation, and a small structured `{ error: ... }` JSON body for API/fetch clients — e.g. a stale `/api/v1/asset_downloads/:id/download` inbox link no longer surfaces a stack trace.
| 📋 Planned | Image/video coordinate annotations | Draw shapes or drop a pin comment at a specific image coordinate or video timestamp (distinct from the existing workflow approval/rejection note field, which is the only "comment" mechanism today).
| 📋 Planned | General asset commenting | A standalone comment thread on an asset (open to any collaborator, not tied to a workflow approval step).
| 📋 Planned | UI extension/plugin framework | A documented hook/widget API so third parties can add custom panels, dashboard widgets, or Properties-page tabs without forking the React source (custom *metadata fields* are already fully supported via Metadata Schemas — this item is about extending the *UI itself*, e.g. new widget types beyond the single built-in `BarChartWidget`).
| 📋 Planned | Distribution portal for external stakeholders | Expand today's signed, time-expiring Collection share links into a full-fledged branded portal with per-asset (not just per-collection) permissions and download tracking for external reviewers/partners.
| 📋 Planned | Rights & licensing enforcement | Turn today's rights *metadata* (`usage_terms`, `license_expires_at` on Asset; compliance-violation checks on Collections) into enforced behaviour — e.g. blocking download/export of an expired-license or internal-only asset, plus a licensing-terms management UI.
|===

[[aem-parity-checklist]]
=== Capability parity checklist (vs. a typical enterprise DAM suite)

The table below was produced by directly auditing this codebase (models,
controllers, GraphQL schema, and React components — not assumptions) against
a checklist of capabilities commonly expected of an enterprise DAM/Assets
product. It is meant to answer "is X implemented, partial, or not built yet"
for planning purposes. Genuine gaps are already reflected as
📋 Planned rows in the <<module-roadmap,product roadmap>> above.

[cols="3,1,5", options="header"]
|===
| Capability | Status | Notes / evidence

| Create folders to organize assets | ✅ Done | Hierarchical `Folder` model (self-referential nesting), full CRUD, breadcrumbs, soft-delete.
| Upload assets | ✅ Done | Single + bulk drag-drop upload, duplicate detection, schema assignment at upload time.
| Publish assets | ✅ Done | `Asset#status` still includes the older workflow-driven `ready`/`approved` states, but publishing is now also a direct, one-click action independent of any workflow: `POST/POST /api/v1/assets/:id/publish`\|`/unpublish` (immediate or scheduled for a future date/time via `ScheduledPublishAction`/`PublishSchedulerWorker`), surfaced through the "Manage Publish" menu next to *Tools*.
| Edit, move, and copy assets | ✅ Done | Edit, move, and copy are all full REST actions (`Api::V1::Assets#update`, `Api::V1::MoveOperations#create`, `Api::V1::CopyOperations#create`). Copy recursively duplicates folders (subfolders + assets), shares the underlying ActiveStorage blob rather than re-uploading file bytes, and auto-resolves destination name collisions.
| Browse, filter, and search assets, including similarity search | ✅ Done | Full-text + faceted REST search, pgvector cosine-similarity semantic/visual search, agentic AI Copilot search mode.
| Add/edit Basic-tab metadata field values (except Smart Tags) | ✅ Done | Schema-driven `AssetMetadataPanel` with per-field `read_only` support; AI-generated tags are shown read-only and separated from manually-editable static tags.
| Add and delete static renditions | ❌ Not built | `Rendition` model/table exist but are worker-generated only (video MP4/OGG/AV1); no user-facing add/delete UI or endpoint yet (tracked above).
| Download folders, assets, and asset renditions | ✅ Done | Individual asset + individual rendition download; bulk folder/asset ZIP export via `AssetDownloadWorker` with progress polling, queue-awareness, and inbox/notification delivery of the finished link (`Api::V1::AssetDownloadsController`).
| Create asset versions | ✅ Done | Full immutable version history per asset with restore-to-prior-version support.
| Create and perform review tasks on assets | ✅ Done | Workflow approval nodes with configurable assignees/any-all logic, SLA deadlines, and a Workflow Dashboard of pending/active/completed tasks.
| Annotate assets (coordinate/timestamp pins) | ❌ Not built | No spatial/timestamp annotation model or UI exists yet (tracked above). Only a static CONFIDENTIAL watermark overlay exists, which is unrelated.
| HTTP REST and GraphQL APIs (for assets/folders generally) | ✅ Done | ~44 REST controllers plus a full GraphQL schema (queries, 33+ mutations) covering assets, folders, collections, and search.
| Marketing Cloud integration | ➖ N/A | Adobe Experience/Marketing Cloud–specific. No Salesforce Marketing Cloud or Adobe Experience Cloud connector exists, and none is currently planned — Capri DAM's integration surface is its own REST/GraphQL API plus the 13 ingestion connectors below.
| Customize and extend the asset management UI | 🚧 Partial | Custom **metadata fields** are fully supported (Metadata Schemas admin tool). A general UI plugin/widget-extension framework (custom dashboard widgets, custom Properties tabs) does not exist yet (tracked above).
| Query Builder API to extend search | 🚧 Partial | REST search accepts arbitrary metadata/property query params, and GraphQL `searchAssets` accepts a `metadataFilters` map — but there's no graphical query-builder UI and no complex nested boolean-query support.
| Create static tags | ✅ Done | Manual tag editor (`AssetTagsEditor`) with autocomplete, kept separate from AI-suggested tags; workflow nodes can also add/remove tags automatically.
| Author projects and tasks | ✅ Done | `Collection` model serves as the project/campaign workspace (manual or rule-based "smart" membership, group ACLs, expiry); Workflow Tasks provide the task layer.
| Activity stream (timeline) | ✅ Done | Immutable, DB-trigger-protected `AuditLog` plus per-asset usage-event tracking (views/downloads/shares), surfaced in the asset's Audit tab.
| Comments and annotations | 🚧 Partial | The only "comment" mechanism today is the approve/reject note on a workflow task — there's no standalone, general-purpose comment thread on an asset, and no spatial annotation (tracked above, same gap as "Annotate assets").
| Doorkeeper OAuth2 / API auth for integrations | ✅ Done | Full OAuth2 (Doorkeeper) plus GraphQL-managed Personal Access Tokens for third-party API clients.
| Third-party ingestion/migration connectors | ✅ Done | 13 source connectors (AEM, Aprimo, Brandfolder, Bynder, Canto, Cloudinary, Extensis, FTP, Media Valet, Nuxeo, S3, SharePoint, Widen) with encrypted credentials.
| Distribution portal for external stakeholders ("Content Hub") | 🚧 Partial | Collections can be shared externally via signed, time-expiring links today; a full branded, per-asset-permission portal is not built yet (tracked above).
| Dynamic Media (image/video transformation & delivery) | ✅ Done | Real-time edge CDN delivery, smart subject-aware cropping, multi-codec video encoding (H.264/AV1/VP9), HLS/DASH streaming, signed URLs, per-folder delivery profiles. (AVIF output format is the one piece still 🚧 in progress — see the roadmap above.)
| Rights management / licensing | 🚧 Partial | Assets store usage-terms and license-expiry metadata, and Collections flag compliance violations — but nothing is *enforced* yet (e.g. an expired-license asset can still be downloaded), and there's no licensing-terms management UI (tracked above).
|===

[[test-coverage-roadmap]]
=== Consolidated test-coverage roadmap

This guide's screenshots and feature descriptions are accurate as of today, but
**E2E test coverage across the product is uneven** — some modules (Recycle Bin,
Access Control) are exercised extremely thoroughly, while others (Delivery & CDN,
System Operations sub-tabs, several AI features) have little to none. The table
below consolidates every `[.status-pending]` item called out per-module into a
single backlog, to help prioritise future E2E test-writing.

[cols="2,4", options="header"]
|===
| Module | Pending E2E scenarios

| Dashboard | (see <<module-dashboard>>) — widget-level interaction/drill-down behaviour.
| Asset Management | (see <<module-asset-management>>) — version history diff view, bulk metadata CSV import/export.
| Search & Discovery | (see <<module-search>>) — semantic search relevance validation, semantic cluster map interactions.
| Recycle Bin | Restoring an item; permanently purging via Empty Bin; workflow-protected purge guard; retention-policy save.
| Workflow Automation | Parallel/Sequential Review and Slack/In-App Alert nodes; full workflow execution (trigger → task → resolution); Workflow Dashboard screen; editing a published blueprint.
| Access Control & Security | OAuth2 (Doorkeeper) token issuance; server-side enforcement of the 7-column ACL; full Keycloak OIDC round-trip.
| AI Intelligence Suite | AI Copilot chat interaction; Agent Automations; Batch Processing; Prompt Playground; creating a model endpoint/style preset; C2PA record signing.
| Email / Communication Engine | Real email send/receive round-trip; event-mapping creation; AI Suggestions generation; Outbox filtering.
| Ingestion & Migration | Completing the migration wizard against a real connector; Legacy Connectors screen; batch execution monitoring; per-source-platform behaviour.
| System Operations & Admin | ✅ Now covered: Observability tab (vitals + restart flow), Operational Logging save flow (incl. TRACE level), AI Gateway controls (simulated UI), Feature Flags placeholder, Reports screen (Analytics Dashboard, Download Center, Report Types, Create Export drawer). Remaining gap: Storage & Edge save flow is covered only for the Fastly provider — Cloudflare/Akamai provider switch is still untested.
| Localization / i18n | E2E coverage for the remaining 7 non-German locales; RTL layout verification for Arabic.
| Delivery & CDN | **Zero E2E coverage today** — transform correctness, smart-crop, HMAC signing, cache purge, video encoding, profile assignment.
|===

[.feature-highlight]
====
**Suggested priority order** for closing these gaps: (1) Delivery & CDN — zero
coverage on a customer-facing, revenue-relevant capability; (2) Workflow
execution end-to-end — high business criticality, currently only the
*designer* is tested, not *execution*; (3) AI Copilot interaction — flagship
feature, currently untested beyond page load; (4) Ingestion & Migration real
connector/batch-execution flows.
====
