This guide collects **operational, step-by-step deployment playbooks** for running Capri DAM in
production on **Amazon Web Services**, using managed services in place of the self-hosted
Postgres/Redis containers used by the <<section-deployment-view,architecture deployment view>>'s
Kamal baseline. It targets **ECS on Fargate** (serverless containers, no EC2 fleet to patch) fronted
by an **Application Load Balancer**, with **Aurora PostgreSQL** (pgvector-compatible),
**ElastiCache for Redis**, and **S3** replacing the Kamal accessories and local-disk storage.

This is a sibling guide, not a replacement — see link:../deployment-guide-kamal/[the Kamal Deployment
Guide] for the plain-Linux-server/VPS target, and
link:../deployment-guide-azure-kubernetes/[the Azure/Kubernetes Deployment Guide] for AKS. Pick
whichever matches your team's existing cloud footprint and operational preferences; the application
code and Docker image are identical across all three.
