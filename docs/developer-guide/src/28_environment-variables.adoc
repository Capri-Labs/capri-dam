[[section-environment-variables]]
== Environment Variables Reference

This is the single canonical list of every environment variable Capri DAM reads at boot or at
runtime, consolidated from `config/database.yml`, `config/environments/production.rb`,
`config/initializers/*.rb`, `config/deploy.yml`, and `config/puma.rb`. Each of the three deployment
guides (link:../deployment-guide-kamal/[Kamal], link:../deployment-guide-aws/[AWS],
link:../deployment-guide-azure-kubernetes/[Azure/Kubernetes]) wires these into its own
secret-management mechanism (Kamal `env.secret`, AWS Secrets Manager, Azure Key Vault) — this chapter
documents *what* each variable does, independent of *how* it's injected.

=== Required in every environment

[cols="1,1,3", options="header"]
|===
| Variable | Secret? | Purpose
| `RAILS_MASTER_KEY` | Yes | Decrypts `config/credentials.yml.enc` at boot. Without it the app cannot start in any environment where `config/master.key` isn't present on disk (i.e. every deployed environment). Never bake into an image or commit it.
| `DATABASE_URL` (or `DB_HOST`/`DB_PORT` + `HEADLESS_DAM_DATABASE_PASSWORD`) | Yes (password) | PostgreSQL connection. `config/database.yml` supports either a full `DATABASE_URL`, or split `DB_HOST`/`DB_PORT` (default `localhost`/`5432`) with the password read from `HEADLESS_DAM_DATABASE_PASSWORD`. The target Postgres must have the `vector` extension available (see each deployment guide's "Managed Data Services"/"Accessories" chapter).
| `REDIS_URL` | Depends on target | Backs Sidekiq, Action Cable (`config/cable.yml`), and rate limiting. Defaults to `redis://localhost:6379/0` in development; **must** be set explicitly in every deployed environment (`config/initializers/sidekiq.rb`, `system_config_subscriber.rb`).
| `COVERBAND_REDIS_URL` | Depends on target | Separate Redis logical DB (or endpoint) for Coverband code-coverage tracking in production — kept distinct from the main `REDIS_URL` so coverage data doesn't compete with job-queue traffic.
|===

=== Application configuration

[cols="1,1,3", options="header"]
|===
| Variable | Default | Purpose
| `RAILS_ENV` | — | Standard Rails environment selector (`production`, `staging`, `test`, `development`).
| `RAILS_LOG_LEVEL` | `info` | Log verbosity in production (`config/environments/production.rb`).
| `RAILS_LOG_TO_STDOUT` | — (falsy) | When `true`, logs go to STDOUT instead of a file — required for every containerised target (Kamal, ECS, AKS) so the platform's log collector (journald/CloudWatch/Container Insights) can capture output.
| `RAILS_SERVE_STATIC_FILES` | — (falsy) | When `true`, Rails/Thruster serves precompiled assets directly — used in all three deployment guides since there's no separate CDN/static-file tier in front of the app by default.
| `APP_HOST` | `example.com` | Public hostname used for mailer link generation (`action_mailer.default_url_options`) and CORS/host-authorization checks. Must match the domain configured in each deployment guide's SSL/Ingress/ALB chapter.
| `CDN_BASE_URL` | — | Base URL for the CDN edge in front of Active Storage assets (see the Developer Guide's Storage Backends & CDN chapter). Optional — only needed if a CDN is fronting asset delivery.
|===

=== Background jobs & server tuning

[cols="1,1,3", options="header"]
|===
| Variable | Default | Purpose
| `WEB_CONCURRENCY` | — | Number of Puma worker processes (`config/puma.rb`) — tune to available CPU cores.
| `RAILS_MAX_THREADS` | `5` | Puma threads per worker **and** the ActiveRecord connection pool size (`config/database.yml`) — keep these aligned or you'll starve the DB pool under load.
| `RAILS_MIN_THREADS` | — | Puma minimum thread pool size.
| `PUMA_WORKER_TIMEOUT` | — | Custom worker timeout (`config/puma.rb`) for long-running requests (e.g. large asset uploads/exports).
| `PORT` | `3000` | Port Puma binds to inside the container — the Kamal/ECS/AKS health checks and load balancer target groups all assume this matches the container's exposed port.
|===

=== Authentication & SSO

[cols="1,1,3", options="header"]
|===
| Variable | Default | Purpose
| `KEYCLOAK_URL` | `http://localhost:8080` | Base URL of the Keycloak realm used for OmniAuth SSO (`config/initializers/devise.rb`).
| `KEYCLOAK_REALM` | `master` | Keycloak realm name.
| `KEYCLOAK_CLIENT_ID` | `development_id` | OAuth2 client ID registered in Keycloak for this app.
| `KEYCLOAK_CLIENT_SECRET` | `development_secret` | OAuth2 client secret — always override the default in any real deployment; treat as a secret (Secrets Manager/Key Vault/Kamal `env.secret`).
|===

=== Third-party integrations (optional)

[cols="1,3", options="header"]
|===
| Variable | Purpose
| `AI_GATEWAY_URL` / `GATEWAY_SECRET` | Endpoint + shared secret for the AI Gateway used by smart-upload/tagging features (see `docs/ai-gateway-prompt.md`). Omit entirely to disable AI-assisted features.
| `MAILER_SENDER_ADDRESS` | From-address used by ActionMailer for outbound notification emails.
| `TWILIO_ACCOUNT_SID` / `TWILIO_AUTH_TOKEN` / `TWILIO_FROM_NUMBER` | SMS notification integration — only required if SMS notifications are enabled in the Notifications module.
| `OTEL_TRACES_EXPORTER` | OpenTelemetry tracing exporter selection, if distributed tracing is enabled.
|===

=== ActiveRecord encryption (optional, recommended for production)

[cols="1,3", options="header"]
|===
| Variable | Purpose
| `ACTIVE_RECORD_ENCRYPTION_PRIMARY_KEY` | Primary key for ActiveRecord's built-in attribute encryption (used for sensitive model attributes beyond `credentials.yml.enc`-managed secrets).
| `ACTIVE_RECORD_ENCRYPTION_DETERMINISTIC_KEY` | Deterministic-encryption key (enables equality queries on encrypted columns).
| `ACTIVE_RECORD_ENCRYPTION_KEY_DERIVATION_SALT` | Salt used in ActiveRecord's key derivation.
|===

Generate all three with `bin/rails db:encryption:init` and store them the same way you store
`RAILS_MASTER_KEY` — never commit them, never bake them into an image.

=== Storage backend — configured in-app, not via environment variables

Unlike every variable above, **which storage backend is active (local disk, S3, Azure Blob, GCS) is
configured entirely through the in-app Settings UI** (`/admin/settings` → Storage Backend), backed by
the `Setting` model — not by environment variables. This is intentional: it lets an administrator
switch storage backends without a redeploy. See:

* link:../developer-guide/06_storage.html[Developer Guide — Storage Backends & CDN] for the adapter
  architecture (`StorageAdapters::S3Adapter`, `StorageAdapters::AzureAdapter`, etc.).
* Each deployment guide's "Managed Data Services"/"Accessories" chapter for the cloud-specific setup
  required *before* switching the backend in-app (creating the S3 bucket/IAM policy, or the Blob
  Storage account/workload identity).

=== Cross-reference: which guide sets what

[cols="1,1,1,1", options="header"]
|===
| Variable | Kamal guide | AWS guide | Azure/K8s guide
| `RAILS_MASTER_KEY` | `env.secret` in `config/deploy.yml` | Secrets Manager → ECS `secrets` | Key Vault (CSI) or `Secret` → pod `env`
| `DATABASE_URL`/`DB_HOST` | Kamal accessory hostname | Secrets Manager (Aurora endpoint) | Key Vault/`Secret` (Flexible Server FQDN)
| `REDIS_URL` | Kamal accessory hostname | Secrets Manager (ElastiCache endpoint) | Key Vault/`Secret` (Azure Cache endpoint)
| Storage backend | Local disk (bind mount) or switch to S3/Azure in-app | S3 (in-app switch) | Azure Blob (in-app switch)
|===
