[[section-ui-components]]
= UI Components

The React frontend is organized into domain folders under
`app/javascript/components/`. Complex modules are mounted as islands inside the
Rails shell and typically fetch JSON from `/api/v1/...`.

== Core library screens

[cols="1,3", options="header"]
|===
| Component area | Notes
| `Folders/*` | Asset explorer, folder tree, upload flows, and the image editor dialog.
| `Search/*` | Full-text and semantic search experiences layered on asset APIs.
| `Dashboard/*` | Top-level KPI and activity screens.
| `Bin/*` | Dedicated recycle-bin manager with stats, filtering, and mixed item restore/delete.
| `Collections/*` | Workspace board, detail view, rule simulation, and semantic cluster map.
| `Duplicates/*` | Duplicate-review dashboard with modal resolution workflow.
| `NotificationBell.jsx` | Global unread bell polling `/api/v1/notifications`.
|===

== Admin surfaces

[cols="1,3", options="header"]
|===
| Component | Responsibility
| `Admin/UsersManager.jsx`, `UserDrawer.jsx` | User CRUD, role changes, membership, and account detail.
| `Admin/UserGroupsManager.jsx` | Hierarchy management for nested groups.
| `Admin/AclMatrix.jsx` | Folder-policy matrix editing against the current `read_access`/`modify_access` style columns.
| `Admin/EmailEngineManager.jsx` | Template editing, event mapping, and outbox monitoring.
| `Admin/SystemStatus/index.jsx` | Operational status tabs for AI gateway, SMTP, observability, feature flags, and storage operations.
| `Admin/Reports/AnalyticsDashboard.jsx` | Analytics charts and export launch point.
| `Admin/Migrations/*` | Connectors, ingestion dashboard, data-health review, and pre-flight tooling.
| `Admin/Intelligence/*` | Model hub, provenance policy, batch AI jobs, prompt playground, semantic copilot, and agent workflows.
|===

== Global layout (Header & Footer)

[cols="1,3", options="header"]
|===
| Component | Notes
| `Layout/Header.jsx` | Fixed top `AppBar` — product logo (`/images/logo.png`), global search bar, quick-create menu, notification bell, and the profile/impersonation menu.
| `Layout/Footer.jsx` | Global site footer, redesigned to match the product logo/brand gradient (indigo `#4f46e5` \-> violet `#7c3aed`, shown as a thin accent stripe across the top of the footer). Renders the product logo (scaled down), a tagline, and three link columns: *Product* (`/dashboard`, `/search`, `/workflows`, `/duplicates`), *Resources* (`/api/rest`, `/api/graphql`, `/api-docs`), and *Legal & Security* (`/privacy`, `/terms`, `/compliance`). The bottom bar shows copyright, an "All systems operational" status dot, and the build version. Fully i18n'd via the `footer.*` translation namespace in `app/javascript/i18n/locales/*.json` — no literal English strings are hard-coded in JSX.
| `Layout/ImpersonationBanner.jsx` | Warning banner rendered above the `AppBar` while an admin is impersonating another user.
|===
