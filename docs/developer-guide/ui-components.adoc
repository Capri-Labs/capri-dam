[[section-ui-components]]
= UI Components

The frontend is a collection of **React 19** islands mounted by Hotwire/Turbo onto
server-rendered HTML shells.  All components live under
`app/javascript/components/<Domain>/` and use **Material UI v9** with `slotProps`
for all slot customisation (no deprecated prop aliases).

Translations use `useTranslation()` from `react-i18next`;
all strings are in `app/javascript/i18n/locales/en.json` (and mirrored to 8 other locales).

== Entry points

=== `app/javascript/application.js`

Mounts React roots after Turbo page loads:

[source,javascript]
----
import { Header }  from './components/Layout/Header';
import { Sidebar } from './components/Layout/Sidebar';
// ...
document.addEventListener('turbo:load', () => {
  mountRoot('capri-header',  <Header />, NotificationProvider);
  mountRoot('capri-sidebar', <Sidebar />);
  mountRoot('capri-footer',  <Footer />);
  // domain-specific roots are mounted by their Turbo frames
});
----

=== `app/javascript/components/App.jsx`

Main layout shell wrapping the drawer nav and routing view.
Renders either `AssetExplorer` (default) or `WorkflowDashboard` depending on the
active route.

== i18n

All 9 locale files share the same key structure:

[source,json]
----
{
  "common": { "save": "Save", "cancel": "Cancel", ... },
  "assets": { "upload": "Upload", "metadata": "Metadata", ... },
  "workflows": { "approve": "Approve", "reject": "Reject", ... }
}
----

Add new keys to `en.json` first, then the other 8 files
(`de`, `es`, `fr`, `ja`, `ko`, `nl`, `pt`, `zh`).

== MUI v9 conventions

Use `slotProps` everywhere:

[source,jsx]
----
// ✅ correct (MUI v9)
<TextField slotProps={{ input: { startAdornment: <SearchIcon /> } }} />

// ❌ deprecated
<TextField InputProps={{ startAdornment: <SearchIcon /> }} />
----

Run `script/mui_v9_migrate.py` to auto-migrate older components if needed.
