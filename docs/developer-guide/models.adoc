[[section-models]]
= Domain Models

Capri DAM follows **Active Record** conventions with all models in `app/models/`.
Business logic lives in service objects (`app/services/`); models stay lean.

== Core Entities

=== Asset (`app/models/asset.rb`)

The central entity representing a managed digital file.

[cols="2,1,4", options="header"]
|===
| Column / Association | Type | Description

| `title` | string | Human-readable name (defaults to filename).
| `description` | text | Free-text description.
| `status` | enum | `active`, `archived`, `quarantined`.
| `file` (ActiveStorage) | attachment | The master binary — stored in the configured backend.
| `content_type` | string | MIME type detected on upload.
| `file_size` | integer | Byte count of the master file.
| `checksum` | string | SHA-256 of the master binary for deduplication.
| `metadata` | jsonb | EXIF/IPTC/XMP technical metadata, extracted by `ExtractionWorker`.
| `custom_metadata` | jsonb | Schema-driven business metadata (see `MetadataSchema`).
| `focal_point_x/y` | float | Normalised 0–1 focal point used by Fastly smart-crop.
| `folder` | belongs_to | Parent `Folder`.
| `created_by` | belongs_to User | Uploader.
| `versions` | has_many AssetVersion | Immutable version history.
| `embeddings` | has_one AssetEmbedding | pgvector embedding for semantic search.
| `collections` | many-through | Membership via `CollectionAsset`.
| `workflow_instances` | has_many | Active workflow runs for this asset.
| `deleted_at` | datetime | Soft-delete timestamp (via `SoftDeletable` concern).
|===

.Key scopes
[source,ruby]
----
Asset.active          # where(deleted_at: nil, status: :active)
Asset.in_bin          # soft-deleted (deleted_at IS NOT NULL)
Asset.images          # content_type ILIKE 'image/%'
Asset.videos          # content_type ILIKE 'video/%'
Asset.search(query)   # full-text via PG tsvector
----

.Concerns included
* `Auditable` — writes to `audit_logs` on create/update/destroy.
* `SoftDeletable` — `trash!` / `restore!` / `permanently_delete!` methods.

=== Folder (`app/models/folder.rb`)

N-ary tree using **parent_id** self-referential association with a separate
`UserGroupClosure` table tracking ancestry.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `name` | string | Display name.
| `slug` | string | URL-safe segment (unique within parent).
| `path` | string | Full materialised path, e.g. `marketing/campaigns/q1`.
| `parent` | belongs_to Folder | `nil` for root folders.
| `children` | has_many Folder | Direct sub-folders.
| `assets` | has_many Asset | Files living directly in this folder.
| `governing_groups` | has_many UserGroup | Groups whose ACL policy applies.
| `metadata_schema` | through assignment | Active `MetadataSchema` for this path.
| `image_profile` | through assignment | Active `ImageProfile` for this path.
| `video_profile` | through assignment | Active `VideoProfile` for this path.
| `folder_policy` | has_one FolderPolicy | RBAC override for this specific folder.
| `deleted_at` | datetime | Soft-delete.
|===

=== User (`app/models/user.rb`)

Devise model with Keycloak SSO support and a group-based RBAC permission system.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `email` | string | Primary identifier, unique.
| `role` | enum | `viewer`, `contributor`, `manager`, `admin`, `super_admin`.
| `provider` / `uid` | string | OmniAuth SSO identity.
| `groups` | many-through | `UserGroupMembership` → `UserGroup`.
| `preferences` | has_one UserPreference | Notification flags, locale, theme.
| `personal_access_tokens` | has_many | API token management.
| `impersonator` | has_one UserImpersonator | Active impersonation session.
| `deleted_at` | datetime | Soft-delete (retains audit trail).
|===

.Permission resolution
[source,ruby]
----
user.can?(:upload, folder)   # aggregates group ACLs + role + FolderPolicy
user.effective_permissions   # returns hash of all folder ACL columns
user.all_group_ids           # includes inherited group memberships via closure table
----

=== UserGroup (`app/models/user_group.rb`)

Closure-table hierarchy (`UserGroupClosure`) enabling nested group inheritance.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `name` | string | Display name.
| `parent` | belongs_to UserGroup | `nil` for top-level groups.
| `members` | many-through UserGroupMembership | Direct users.
| `folder_policies` | has_many FolderPolicy | ACL overrides assigned to this group.
|===

=== FolderPolicy (`app/models/folder_policy.rb`)

7-column ACL matrix controlling per-group access to a folder.

[cols="1,1,4", options="header"]
|===
| Column | Type | Description

| `user_group` | belongs_to | Target group.
| `folder` | belongs_to | Target folder.
| `can_read` | boolean | Browse / view assets.
| `can_upload` | boolean | Upload new assets.
| `can_edit_metadata` | boolean | Edit metadata fields.
| `can_download` | boolean | Download master files.
| `can_delete` | boolean | Trash assets.
| `can_manage` | boolean | Change folder settings and policies.
| `can_share` | boolean | Generate public / signed share URLs.
|===

== Media & Metadata

=== AssetVersion (`app/models/asset_version.rb`)

Immutable version snapshot created every time an asset is replaced.

=== AssetEmbedding (`app/models/asset_embedding.rb`)

Stores the `pgvector` float vector for semantic similarity search.
Generated by `ExtractionWorker` using the configured AI provider.

=== MetadataSchema (`app/models/metadata_schema.rb`)

Configurable field definitions assigned to folder paths.
Each entry has a `fields` JSONB array:

[source,json]
----
[
  { "key": "campaign", "label": "Campaign", "type": "string", "required": true },
  { "key": "approved", "label": "Approved", "type": "boolean" }
]
----

=== Rendition (`app/models/rendition.rb`)

A pre-generated binary rendition (thumbnail, web-optimised) stored in
ActiveStorage. Most delivery uses on-the-fly Fastly IO instead.

=== TransformationPreset (`app/models/transformation_preset.rb`)

Named Fastly IO URL parameter sets (width, height, crop, format, quality)
referenced by `ImageProfile`.

=== ImageProfile (`app/models/image_profile.rb`)

Named profile bundling `TransformationPreset` entries assigned to folder paths
via `ImageProfileFolderAssignment`.

=== VideoProfile (`app/models/video_profile.rb`)

Encoding profile (codec, bitrate, resolution) assigned to folder paths via
`VideoProfileFolderAssignment`.  Transcoding is queued in Sidekiq.

== Collections

=== Collection (`app/models/collection.rb`)

A curated or auto-populated (smart) set of assets.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `name` | string | Display name.
| `collection_type` | enum | `manual`, `smart`, `temporal`.
| `rules` | has_many CollectionRule | Smart-collection filter rules.
| `assets` | many-through CollectionAsset | Members for manual collections.
| `owner` | belongs_to User | Creator.
|===

=== CollectionRule (`app/models/collection_rule.rb`)

Predicate for smart collections: `{ field, operator, value }`.

== Background & Ingestion

=== IngestionBatch (`app/models/ingestion_batch.rb`)

Tracks a bulk-import job from an external DAM or storage source.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `source_type` | string | Adapter name (e.g. `s3`, `nuxeo`, `brandfolder`).
| `status` | enum | `pending`, `running`, `completed`, `failed`.
| `items` | has_many IngestionItem | Per-file records.
| `config` | jsonb | Source-specific credentials / options (encrypted at rest).
|===

=== IngestionItem (`app/models/ingestion_item.rb`)

One file in an `IngestionBatch`. Transitions: `pending → processing → done / failed`.

== Reporting & Audit

=== AuditLog (`app/models/audit_log.rb`)

Immutable event record written by the `Auditable` concern.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `user` | belongs_to | Actor (nullable for system events).
| `action` | string | `create`, `update`, `destroy`, `download`, `share`.
| `auditable` | polymorphic | Target record (Asset, Folder, …).
| `metadata` | jsonb | Before/after diff or context payload.
| `ip_address` | string | Originating IP.
|===

=== DailyMetric / ReportDefinition / ReportSnapshot

Aggregated statistics collected by `Metrics::Aggregator` (runs at 01:00 UTC).
`ReportDefinition` stores a report configuration; `ReportSnapshot` stores the
generated output (CSV / XLSX / PDF via ActiveStorage).

== Workflow

=== Workflow / WorkflowInstance / WorkflowStep / WorkflowTask

[cols="1,4", options="header"]
|===
| Model | Purpose

| `Workflow` | Template definition (DAG of steps) built in the Workflow Designer UI.
| `WorkflowInstance` | A live execution attached to an asset or collection.
| `WorkflowStep` | A node in the template DAG (approval, notification, auto-action).
| `WorkflowTask` | A human task created for a specific user/group during a run.
|===

== System & Configuration

=== StorageBackend (`app/models/storage_backend.rb`)

Runtime-configurable storage connection (name, adapter, credentials JSONB, active flag).
Adapters: `local`, `s3`, `gcs`, `azure`, `wasabi`, `r2`, `spaces`, `backblaze`.

=== CdnConfiguration (`app/models/cdn_configuration.rb`)

CDN provider settings. Adapters: `fastly`, `cloudflare`, `akamai`.

=== SystemConfiguration (`app/models/system_configuration.rb`)

Key-value store for global runtime settings (max upload size, AI provider, etc.).

=== SystemConnector (`app/models/system_connector.rb`)

External integration endpoint configuration (webhook target, OAuth2 credentials).

=== AiConfiguration (`app/models/ai_configuration.rb`)

AI/ML provider settings used by `AssetEmbedding` generation and AI Copilot features.

=== Notification (`app/models/notification.rb`)

In-app notification record delivered to a user (polymorphic `notifiable` target).

=== PersonalAccessToken (`app/models/personal_access_token.rb`)

Hashed API token for programmatic access. Replaces session auth for CI/API clients.

=== QuarantinedAsset (`app/models/quarantined_asset.rb`)

Reference to an asset flagged by the `SecurityAnomalyDetector` (malware scan, policy violation).
