[[section-workers]]
= Background Workers & Jobs

All async work runs through **Sidekiq** backed by **Redis**.
Workers live in `app/workers/`; configuration in `config/sidekiq.yml`.

Start workers: `bundle exec sidekiq -C config/sidekiq.yml`  (or `make dev`).

== Queue topology

[cols="1,1,3", options="header"]
|===
| Queue | Priority | Work routed here

| `critical` | highest | Workflow tasks, notifications, impersonation audit events.
| `default` | normal | Asset processing, extraction, embedding generation, CDN purge.
| `bulk` | low | Ingestion batches, metadata imports/exports, duplicate scans.
| `mailers` | low | Email delivery via `EmailDispatcherWorker`.
| `reports` | low | Report generation (CSV/XLSX/PDF).
| `maintenance` | lowest | Cleanup and purge jobs.
|===

== Worker catalogue

=== Asset processing

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `AssetProcessorWorker` | Entry point after upload. Runs EXIF/IPTC extraction, generates thumbnail rendition, queues `ExtractionWorker` and `DuplicateDetectionWorker`.
| `ExtractionWorker` | Reads binary metadata (EXIF/IPTC/XMP) and updates `asset.metadata`. Triggers `AssetEmbeddingWorker` if AI is configured.
| `AssetWorkflowTriggerWorker` | Checks if any `Workflow` rule matches the new asset and starts a `WorkflowInstance`.
|===

=== Search & AI

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `FolderMetadataSyncWorker` | Syncs folder path metadata to the search index after folder changes.
| `EdgeMetadataSyncWorker` | Pushes asset metadata updates to Fastly edge dictionaries for header-based delivery rules.
|===

=== Ingestion

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `IngestionWorker` | Processes a single `IngestionItem`: downloads from source, uploads to ActiveStorage, creates `Asset`.
| `PreFlightAnalysisWorker` | Analyses the source manifest before ingestion to detect naming conflicts and file-type violations.
| `MigrationCommitWorker` | Atomically copies existing blobs from one `StorageBackend` to another during provider migrations.
| `MigrationTransformWorker` | Optional transformation pass during migration (re-compress, strip metadata).
| `MigrationReportWorker` | Generates a migration completion report.
|===

=== Metadata

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `MetadataExportWorker` | Generates a CSV of asset metadata for a folder; saves result via ActiveStorage.
| `MetadataExportCleanupWorker` | Purges export CSVs older than 30 days. Runs daily at 02:00 UTC.
| `MetadataImportWorker` | Processes an uploaded CSV row-by-row, updating `asset.custom_metadata`.
| `MetadataImportCleanupWorker` | Purges import artefacts older than 30 days. Runs daily at 02:15 UTC.
|===

=== Duplicates

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `DuplicateDetectionWorker` | Compares a new asset checksum against existing assets; creates/updates `DuplicateGroup`.
| `DuplicateRepositoryScanWorker` | Full-library scan for duplicates. Triggered manually from the Admin UI.
|===

=== Workflow engine

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `WorkflowInitiatorWorker` | Creates a `WorkflowInstance` and the first `WorkflowTask`.
| `WorkflowEngineWorker` | Advances a `WorkflowInstance` after a task decision; evaluates step conditions.
| `TaskNotificationWorker` | Sends in-app + email notification to a `WorkflowTask` assignee.
|===

=== Recycle bin

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `BinPurgeWorker` | Permanently destroys assets whose `deleted_at` exceeds `retention_days`. Runs daily at 03:00 UTC.
| `TrashCleanupWorker` | Removes orphaned ActiveStorage blobs after hard-deletes.
|===

=== CDN & email

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `CdnInvalidationWorker` | Issues Fastly Surrogate Key purge requests for a list of asset IDs.
| `EmailDispatcherWorker` | Sends transactional emails via `EmailOrchestrator` using the configured SMTP/SendGrid/Postmark adapter.
|===

=== Collections

[cols="2,4", options="header"]
|===
| Worker | Responsibility

| `SmartCollectionRouterWorker` | Re-evaluates `CollectionRule` sets after asset metadata changes and updates `CollectionAsset` memberships.
|===

== Cron schedule

Defined in `config/schedule.rb` (Whenever gem format):

[source,ruby]
----
every 1.day, at: "1:00 am"  { runner "Metrics::Aggregator.run_daily_snapshot!" }
every 1.day, at: "2:00 am"  { runner "MetadataExportCleanupWorker.perform_async" }
every 1.day, at: "2:15 am"  { runner "MetadataImportCleanupWorker.perform_async" }
every 1.day, at: "3:00 am"  { runner "BinPurgeWorker.perform_async" }
----

== Adding a new worker

. Create `app/workers/my_thing_worker.rb`:
+
[source,ruby]
----
class MyThingWorker
  include Sidekiq::Worker
  sidekiq_options queue: :default, retry: 3

  def perform(thing_id)
    thing = Thing.find(thing_id)
    # business logic here
  end
end
----

. Add a spec: `spec/workers/my_thing_worker_spec.rb`.
. Enqueue inline: `MyThingWorker.perform_async(thing.id)`.
. For scheduled work, add a line to `config/schedule.rb`.

== Monitoring

Sidekiq Web UI is mounted at `/admin/sidekiq` (admin-role required).
Queue depths and error counts are surfaced on the `GET /admin/system_status` page.
