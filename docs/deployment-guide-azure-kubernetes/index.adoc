This guide collects **operational, step-by-step deployment playbooks** for running Capri DAM on
**Microsoft Azure using Kubernetes** — specifically **Azure Kubernetes Service (AKS)** — as an
alternative to the self-hosted single-VM Kamal target and the AWS ECS target. It replaces the Kamal
guide's self-hosted Postgres/Redis containers with **Azure Database for PostgreSQL — Flexible Server**
(pgvector-compatible), **Azure Cache for Redis**, and **Azure Blob Storage**, deploying the app as
standard Kubernetes `Deployment`/`Service`/`Ingress` manifests (or an equivalent Helm chart).

This is a sibling guide, not a replacement — see link:../deployment-guide-kamal/[the Kamal Deployment
Guide] for the plain-Linux-server/VPS target, and link:../deployment-guide-aws/[the AWS Deployment
Guide] for ECS/Fargate. Pick whichever matches your team's existing cloud footprint and Kubernetes
familiarity; the application code and Docker image are identical across all three.
