== SSL, Domains & Ingress

=== Install ingress-nginx and cert-manager

[source,bash]
----
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm install ingress-nginx ingress-nginx/ingress-nginx --namespace ingress-nginx --create-namespace

helm repo add jetstack https://charts.jetstack.io
helm install cert-manager jetstack/cert-manager --namespace cert-manager --create-namespace \
  --set installCRDs=true
----

This guide uses **ingress-nginx + cert-manager** for portability across clouds (matching the pattern
used by the AWS guide's ALB and the Kamal guide's kamal-proxy). If you prefer Azure-native tooling,
substitute the **Application Gateway Ingress Controller (AGIC)** and **Azure-managed certificates**
instead — the `Ingress` resource shape stays nearly identical, only the `ingressClassName` and
annotations change.

=== `ClusterIssuer` for Let's Encrypt

[source,yaml]
----
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata: { name: letsencrypt-prod }
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: devops@example.com
    privateKeySecretRef: { name: letsencrypt-prod-key }
    solvers:
      - http01:
          ingress: { ingressClassName: nginx }
----

cert-manager watches the `Ingress` resource created in <<section-azure-k8s-cluster-and-manifests>>
(annotated `cert-manager.io/cluster-issuer: letsencrypt-prod`), automatically requests and renews the
certificate, and stores it in the `capri-dam-tls` `Secret` referenced by the `Ingress`'s `tls` block.

=== DNS: pointing your domain at the ingress controller

Find the ingress controller's public IP (provisioned as an Azure Load Balancer by the `ingress-nginx`
Helm chart):

[source,bash]
----
kubectl get service ingress-nginx-controller -n ingress-nginx \
  -o jsonpath='{.status.loadBalancer.ingress[0].ip}'
----

Add an `A` record for `dam.example.com` pointing at that IP — via Azure DNS:

[source,bash]
----
az network dns record-set a add-record --resource-group capri-dam-rg --zone-name example.com \
  --record-set-name dam --ipv4-address "$INGRESS_IP"
----

or the equivalent record at your existing registrar if the zone isn't delegated to Azure DNS.

=== Multiple domains / environments

Add another `host` entry (and, if needed, another `Ingress` resource with a distinct `tls.hosts`) for
e.g. `staging.dam.example.com`, routed to a separate `Service`/`Deployment` pair in the same or a
different namespace — a single ingress controller can front any number of hostnames without
provisioning a new load balancer per environment.
